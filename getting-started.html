Installation Guide

Getting Started

Install Ralph and run your first automated workflow in under 10 minutes

01

Prerequisites

What you need before installing Ralph

Rust Toolchain

Ralph is built in Rust and installed via Cargo. You'll need Rust 1.70 or newer.

Install Rust via rustup.rs

AI Agent

Ralph orchestrates AI agents. Install at least one supported agent:

  • Claude Code — Anthropic's coding agent
  • Codex — OpenAI's code model
  • OpenCode — Open-source alternative

Git

Ralph creates semantic commits automatically. Make sure Git is installed and configured.

git --version
02

Installation

Clone and build Ralph from source

1

Clone the repository

bash 
git clone ssh://git@codeberg.org/mistlight/Ralph-Workflow.git

If SSH isn't set up, use HTTPS: https://codeberg.org/mistlight/Ralph-Workflow.git

2

Navigate to the directory

bash 
cd Ralph-Workflow
3

Install with Cargo

bash 
cargo install --path .

This compiles Ralph and installs it to ~/.cargo/bin/

4

Verify installation

bash 
ralph --version
You should see Ralph's version number. Installation complete!

Troubleshooting

  • "command not found" error?

    Add Cargo's bin directory to your PATH:

    export PATH="$HOME/.cargo/bin:$PATH"
  • Rust not installed?

    Install it from rustup.rs — it takes about 2 minutes.

  • SSH clone fails?

    Either add your SSH key to Codeberg, or use HTTPS clone URL instead.

03

Your First Run

Create a PROMPT.md and let Ralph do the work

Step 1: Create PROMPT.md

In your project's root directory, create a file named PROMPT.md. This is where you describe what you want Ralph to build.

markdown PROMPT.md 
# Add a Hello World Endpoint

## Goal
Add a simple `/hello` endpoint that returns "Hello, World!"

## Requirements
- Create a GET endpoint at `/hello`
- Return plain text response "Hello, World!"
- Add basic error handling

## Constraints
- Follow existing project patterns
- Keep it simple for now

Step 2: Run Ralph

With your PROMPT.md ready, run Ralph in Standard mode:

bash 
ralph -S

Step 3: Watch the Magic

Ralph will:

  1. Read your PROMPT.md
  2. Start the Developer agent to write code
  3. Run the Reviewer agent to check quality
  4. Iterate until the code passes review
  5. Create a semantic Git commit

CLI Quick Reference

-Q
Quick mode (3 iterations)
-S
Standard mode (5 iterations)
-T
Thorough mode (10 iterations)
-L
Long mode (30 iterations)
-q
Quiet mode (minimal output)
-v
Verbose mode (detailed output)

Configuration (Optional)

Customize Ralph's behavior with a config file

Ralph works out of the box, but you can customize its behavior by creating a configuration file.

toml ~/.ralph/config.toml 
# Ralph Configuration
# Create this file at ~/.ralph/config.toml

# Primary agent to use
agent = "claude-code"

# Fallback agents if primary fails
fallback = ["codex", "opencode"]

[reviewer]
# Model for the reviewer agent
model = "claude-sonnet-4"

# Maximum iterations before stopping
max_iterations = 10

[commit]
# Commit message style
style = "conventional"

# Sign commits with GPG
sign = false
