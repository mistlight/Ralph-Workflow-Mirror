Glossary

CLI Terms Made Simple

Essential concepts for anyone new to the command line

Show CLI Glossary

Terminal

What it is: A text-based interface for controlling your computer.

Think of it as: Like having a conversation with your computer using text commands instead of clicking icons.

Example: Instead of double-clicking a file, you type ls to list files.

Command

What it is: A specific instruction you type to tell the computer what to do.

Think of it as: Like clicking a button, but written as text.

Example: ralph -S is a command that runs Ralph in "Short" mode.

PATH

What it is: A list of folders where your computer looks for programs.

Think of it as: Like having a search path—your computer checks these folders when you type a command.

Why it matters: When you install Ralph, it needs to be "in your PATH" so you can run it from anywhere.

Executable

What it is: A file that can run as a program on your computer.

Think of it as: Like an .app file on Mac or .exe on Windows—just a file that does something when you run it.

Example: When you run cargo install, it creates an executable you can run.

Prompt

What it is: The text that appears before the cursor, waiting for you to type something.

Think of it as: Like the "blink cursor" in a text box, showing where your typing will go.

Example: $ is the common prompt symbol, meaning "ready for your command."

Permissions

What it is: Security settings that control what a file or program is allowed to do.

Think of it as: Like "read-only" vs "editable"—permissions tell the computer who can do what.

Why it matters: Sometimes you need to make a file "executable" (allowed to run) with chmod +x.

Don't worry! You don't need to memorize these. Ralph's installation will guide you through everything step by step.

What is PROMPT.md?

Your requirements, AI's implementation

PROMPT.md Markdown 
# Add user authentication

## Requirements
- Email/password login
- Session management
- Password reset flow

## Tech Stack
- Backend: Rust with Actix-web
- Frontend: React + TypeScript
- Database: PostgreSQL

## Notes
- Use JWT for sessions
- Implement rate limiting
- Add unit tests for auth module

The source of truth for your changes

PROMPT.md is a simple Markdown file where you describe what you want to build. Think of it as a ticket, a spec, and a prompt all in one.

  • Write in plain English with Markdown formatting
  • Include requirements, tech stack, and constraints
  • Commit it alongside your changes for full context
  • AI agents read it and iterate until satisfied

Need help? Each step below has detailed instructions.

bash 
git clone ssh://git@codeberg.org/mistlight/Ralph-Workflow.git
cd Ralph-Workflow
cargo install --path .

Requires Rust 1.70+ and cargo in PATH. Get Rust from rustup.rs

The package will be installed as ralph-workflow. Use ralph or ralph-workflow to run.

Verify installation: Run ralph --version to confirm Ralph is installed correctly. You should see version output.

Troubleshooting
  • Command not found? Make sure ~/.cargo/bin is in your PATH. Add this to your shell config: export PATH="$HOME/.cargo/bin:$PATH"
  • Rust not installed? Install via rustup.rs — it's the recommended way
  • SSH clone fails? Make sure your SSH key is added to your Codeberg account, or use HTTPS instead
toml 
# ~/.ralph/config.toml

agent = "claude-code"
fallback = ["codex", "opencode"]

[reviewer]
model = "claude-sonnet-4"
max_iterations = 10

Create this file at ~/.ralph/config.toml

Features

Built for developers who value autonomy and quality

Multi-Agent Orchestration

Developer and Reviewer agents collaborate in real-time. The Developer writes code while the Reviewer validates quality. They iterate automatically until approval.

How the agents work together:

  • Developer Agent: Analyzes your PROMPT.md, breaks down requirements, and implements the code changes. It has full access to your repository and can modify files directly.
  • Reviewer Agent: Examines the code for quality issues, security vulnerabilities, and adherence to best practices. It provides specific feedback when issues are found.
  • Self-Correcting Loop: If the Reviewer finds problems, it sends feedback to the Developer, which fixes them automatically. This continues until both agents agree the code is ready.

Preset Iteration Modes

Five modes from Quick (3 passes) to Long (30 passes). Choose the iteration depth for your task complexity.

Git Integration

Automatic commits with semantic messages. Full history tracking with meaningful descriptions.

Flexible Agent Selection

Claude Code, Codex, or OpenCode. Configure fallback chains for reliability.

Verbosity Control

Quiet mode for CI, full output for debugging, diagnose mode for deep inspection.

Custom Flags

Fine-tune with -D (developer) and -R (reviewer) for exact pass control.

Frequently Asked Questions

Common questions about Ralph Workflow

Getting Started

What AI providers does Ralph support? +

Ralph supports Claude Code, Codex, and OpenCode agents. You can configure a fallback chain in your config file so if one provider fails, Ralph automatically tries the next.

Is Ralph safe to use with my code? +

Yes! Ralph runs entirely locally on your machine. Your code never leaves your computer except when sent to your configured AI provider for processing. You keep full control of your API keys and data.

How do I install Ralph if I don't have Rust? +

You'll need to install Rust first. Visit rustup.rs for a simple installer. Once Rust is installed, you can use cargo install ralph-workflow or build from source.

What is PROMPT.md and how do I create one? +

PROMPT.md is a simple Markdown file in your project directory where you describe what you want to build. Just create a file named PROMPT.md and write your requirements in plain English. Include details like features, tech stack preferences, and constraints. Ralph reads this file and uses it as the source of truth for generating code.

How much does it cost to use Ralph? +

Ralph itself is free and open-source (AGPL-3.0). You only pay for the AI API calls you make to your provider (like Claude or OpenAI). Costs depend on your usage, but typically range from a few cents to a few dollars per task depending on complexity.

Technical

Can Ralph work with my existing codebase? +

Yes! Ralph works with any codebase that uses Git. Simply create a PROMPT.md file in your project root describing the changes you want, and Ralph will read your existing code and make the requested modifications while maintaining your project's structure and conventions.

What if Ralph makes a mistake? +

The Reviewer agent is specifically designed to catch mistakes. It validates code quality, checks for bugs, and requests fixes from the Developer agent until the code meets standards. You can also review the commit before pushing, and Ralph never makes changes to your repository without creating a proper Git commit that you can inspect or revert.

How do I configure API keys? +

Ralph uses your existing AI provider API keys. Configure them in ~/.ralph/config.toml or set them as environment variables. Ralph supports Claude, Codex, and OpenCode agents. See the Advanced install tab for a sample configuration file.

Can I use Ralph in CI/CD? +

Yes! Ralph supports quiet mode (-q) and verbosity control for CI/CD environments. You can run Ralph in automated workflows, GitHub Actions, or any CI/CD pipeline. Configure your API keys as environment variables and run ralph -S -q for silent operation.

New to CLI?

What is the terminal? +

The terminal (also called command line or CLI) is a text-based interface for controlling your computer. Think of it as a direct way to give commands to your operating system. Ralph runs in the terminal, but don't worry—you only need to learn a few simple commands to get started.

What does "PATH" mean? +

PATH is a list of folders your computer searches when you type a command. When you install Ralph with cargo, it gets added to your PATH automatically. This means you can run ralph from anywhere in your terminal without specifying the full file path.

Is Ralph safe for beginners? +

Yes! Ralph is actually great for learning. It runs locally on your machine, so your code never leaves except when sent to your AI provider. Ralph creates git commits before making changes, so you can always see what it did and revert if needed. Think of it as a helpful assistant that shows you exactly what it's doing.

For Developers

How do I configure custom iteration modes? +

Ralph supports preset iteration modes: Quick (3 passes), Standard (5 passes), Thorough (10 passes), Extensive (20 passes), and Long (30 passes). You can also use -D (developer passes) and -R (reviewer passes) flags for fine-grained control. See the Advanced install tab for configuration examples.

How do I configure agent fallback chains? +

In your ~/.ralph/config.toml, specify a primary agent and fallback agents: agent = "claude-code" and fallback = ["codex", "opencode"]. Ralph will automatically try the next agent if one fails, ensuring your workflow continues uninterrupted.

For Vibe Coders

How much time will Ralph save me? +

Most users see tasks completed in 3-10 minutes that would normally take 30-60 minutes of manual back-and-forth with AI chat tools. Ralph eliminates the copy-paste cycle and automatically iterates until code is ready. You can start a task and come back to finished, committed code.

Do I need to be a CLI expert to use Ralph? +

Not at all! Once Ralph is installed (a one-time setup), you only need to remember one command: ralph -S. Write your requirements in plain English in PROMPT.md, run that command, and Ralph handles everything else. It's designed to be simple and hands-free.
