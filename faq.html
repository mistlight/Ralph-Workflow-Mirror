Frequently Asked Questions

Answers to common questions about Ralph Workflow

Getting Started

What is Ralph? +

Ralph is an open-source command-line tool that automates AI coding workflows. Instead of manually copying prompts to an AI chat and pasting code back, Ralph orchestrates two AI agents—a Developer and a Reviewer—that work together in a self-correcting loop until your code is ready.

You write your requirements in a file called PROMPT.md, run ralph -S, and Ralph handles the rest, creating a semantic Git commit when done.

Do I need to know how to code to use Ralph? +

You don't need to be a programmer, but you do need to be comfortable with the command line (terminal). Ralph is designed to be approachable—once installed, you mainly need to know how to write clear requirements in plain English.

If you're new to the terminal, check out our Getting Started guide which walks you through each step.

What is PROMPT.md and how do I create one? +

PROMPT.md is a simple Markdown file in your project directory where you describe what you want Ralph to build. Create a file named PROMPT.md and write your requirements in plain English.

A good PROMPT.md includes:

  • A clear goal (what you want to accomplish)
  • Specific requirements (features, behavior)
  • Constraints (what NOT to do)
  • Success criteria (how to verify it works)

See our Writing Specs guide for templates and examples.

How much does Ralph cost? +

Ralph itself is free—it's open-source under the AGPL-3.0 license.

However, you'll pay for the AI API calls you make to your provider (like Anthropic or OpenAI). Costs depend on your usage:

  • Simple tasks: $0.50 - $2.00
  • Medium tasks: $2.00 - $5.00
  • Complex tasks: $5.00 - $15.00

You can control costs by setting spending limits with your API provider and choosing appropriate iteration modes (-Q for quick, -S for standard).

Installation

What do I need to install Ralph? +

You need:

  • Rust toolchain (version 1.70+) — Install from rustup.rs
  • Git — Usually pre-installed on macOS/Linux
  • An AI agent — Claude Code, Codex, or OpenCode with valid API keys
"Command not found" error after installation? +

This usually means Cargo's bin directory isn't in your PATH. Add it by running:

export PATH="$HOME/.cargo/bin:$PATH"

To make this permanent, add the line to your shell config file (~/.bashrc, ~/.zshrc, etc.) and restart your terminal.

SSH clone fails? +

If you get an SSH error when cloning, you can use HTTPS instead:

git clone https://codeberg.org/mistlight/Ralph-Workflow.git

Or, add your SSH key to your Codeberg account under Settings → SSH / GPG Keys.

Can I install on Windows? +

Yes! Ralph works on Windows. Install Rust via rustup.rs (it will install the MSVC build tools if needed), then follow the same installation steps.

We recommend using Windows Terminal or PowerShell for the best experience.

Usage

What do the different modes mean (-Q, -S, -T, -L)? +

Ralph has preset iteration modes that control how many passes the agents make:

  • -Q Quick — 3 iterations. For simple bug fixes.
  • -S Standard — 5 iterations. Balanced for most tasks.
  • -T Thorough — 10 iterations. For complex features or refactoring.
  • -E Extensive — 20 iterations. For large, difficult tasks.
  • -L Long — 30 iterations. Maximum refinement.

More iterations = higher quality but longer time and higher API costs.

Can Ralph work with my existing codebase? +

Yes! Ralph works with any Git repository. It reads your existing code to understand patterns and conventions, then makes changes while respecting your project structure.

For best results, mention specific files and existing patterns in your PROMPT.md so Ralph knows what to follow.

What if Ralph makes a mistake? +

The Reviewer agent is designed to catch mistakes before code is committed. But if something slips through:

  1. Ralph always creates Git commits, so you can see exactly what changed
  2. You can revert with git reset --hard HEAD~1
  3. Review the diff with git diff HEAD~1 before pushing

Never push code you haven't reviewed. Ralph is a tool, not a replacement for human judgment.

Can I use Ralph in CI/CD pipelines? +

Yes! Ralph supports quiet mode (-q) for CI/CD environments. Configure your API keys as environment variables and run:

ralph -S -q

This suppresses interactive output while still logging to files.

How do I configure API keys? +

Create a config file at ~/.ralph/config.toml:

agent = "claude-code"
fallback = ["codex", "opencode"]

[reviewer]
model = "claude-sonnet-4"
max_iterations = 10

Or set environment variables for your AI provider (e.g., ANTHROPIC_API_KEY).

Troubleshooting

Ralph runs but nothing happens? +

Check these common issues:

  • No PROMPT.md? Make sure you have a PROMPT.md file in your current directory
  • Empty PROMPT.md? The file needs actual requirements, not just a title
  • API key issues? Run with -v (verbose) to see error messages
  • Not in a Git repo? Ralph requires a Git repository to work
API rate limit errors? +

If you hit rate limits:

  • Wait a few minutes and try again
  • Use fewer iterations (-Q instead of -T)
  • Configure fallback agents in your config.toml
  • Check your API provider's rate limit settings
Ralph keeps looping without finishing? +

This usually means the spec is ambiguous or has conflicting requirements. Try:

  • Making your requirements more specific
  • Adding clear acceptance criteria
  • Breaking the task into smaller specs
  • Using a lower iteration mode to see partial results

Licensing

Is Ralph free to use? +

Yes! Ralph is free and open-source under the AGPL-3.0 license. You can use it for personal projects, commercial work, or anything else—no license fee required.

The only cost is the AI API calls you make to your provider.

Does the license affect code Ralph writes for me? +

No. The AGPL-3.0 license applies to Ralph itself—not to the code it generates for your projects.

Your PROMPT.md files and the code Ralph writes belong to you. Ralph is just a tool, like a compiler. The output is yours to license however you choose.

Can I use Ralph for commercial projects? +

Absolutely. The AGPL allows commercial use. You can use Ralph to build commercial software, proprietary applications, or client projects without any licensing fees.

What if I modify Ralph itself? +

If you modify Ralph and distribute it (or run it as a service for others), you must:

  • Keep the AGPL-3.0 license
  • Make your source code available
  • State your changes clearly

See our Open Source page for more details on the license.
